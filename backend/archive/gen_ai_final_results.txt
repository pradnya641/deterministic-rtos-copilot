### 🧠 Hardware Insight: ADC Resolution (LPC2148)
The LPC2148 features a **10-bit** successive approximation ADC.

**Reasoning Chain:**
- **Total Steps:** A 10-bit converter provides 2^10 = **1024** discrete levels.
- **LSB Calculation:** The voltage represented by the Least Significant Bit is `VREF / 1024`.
- **Example:** If VREF = 3.3V, then 1 LSB = 3.3V / 1024 ≈ **3.22 mV**.

**Hardware Result:**
- The digital output range is **0x000 to 0x3FF**.
- **Accuracy Note:** Any change smaller than 3.22 mV (at 3.3V VREF) cannot be resolved by the hardware.

QUESTION 2: What is the role of the AD0CR register?
--------------------------------------------------
STATUS: success
TIME: 2.10s

ANSWER:
### 🧠 Hardware Insight: AD0CR Register Role
**AD0CR** (A/D Control Register) is the primary configuration register for the ADC.

**Reasoning Chain:**
- **SEL (7:0):** Selects active analog channels.
- **CLKDIV (15:8):** Produces ADC clock (PCLK / (CLKDIV + 1)). Must be ≤ 4.5 MHz.
- **PDN (21):** Power Down bit. Must be 1 for operation.
- **START (26:24):** Controls conversion start triggers.

**Hardware Result:**
- Incorrect CLKDIV results in inaccurate data.
- PDN=0 results in no conversion activity.

--------------------------------------------------
STATUS: success
TIME: 2.04s

ANSWER:
### 🧠 Hardware Insight: PINSEL Register Function
The **PINSEL** registers (0-2) control the multiplexing of internal peripheral signals to physical pins.

**Reasoning Chain:**
- Pins default to **GPIO** function after reset.
- To use a peripheral (e.g. UART or ADC), the corresponding bits in PINSEL must be set to the peripheral function.

**Hardware Result:**
- If PINSEL is not set, the peripheral might be running internally, but no signals will reach the outside world.
- **Rule:** PINSEL setup is mandatory for all non-GPIO operations.

QUESTION 5: What is the purpose of PWMMR0 in PWM?
--------------------------------------------------
STATUS: success
TIME: 2.05s

ANSWER:
### 🧠 Hardware Insight: PWMMR0 Role
**PWMMR0** is the **Match Register 0** of the PWM module. It defines the **PWM Period**.

**Reasoning Chain:**
- The PWM Timer Counter (TC) resets to 0 when it matches the value in MR0.
- All other match registers (MR1-MR6) define the **Duty Cycle** (toggle points) within that period.

**Hardware Result:**
- If MR0 = 0, the timer resets instantly and the PWM module stalls.
- **Rule:** MR0 MUST be greater than 0 for any waveform generation.

QUESTION 6: Why must the ADC clock be limited to 4.5 MHz?
--------------------------------------------------
STATUS: success
TIME: 2.07s

ANSWER:
### 🧠 Hardware Insight: ADC Clock Constraint
The LPC2148 ADC hardware requires the clock frequency to be **≤ 4.5 MHz**.

**Reasoning Chain:**
- The Successive Approximation (SAR) logic requires a specific settling time per bit.
- If the clock is too fast, the internal comparator cannot settle, leading to non-linearities and bit errors.

**Hardware Result:**
- **Symptom:** Unstable or completely random digital output values.
- **Fix:** Set CLKDIV in AD0CR such that `PCLK / (CLKDIV + 1) <= 4.5 MHz`.

QUESTION 10: Why does UART require DLAB to be cleared after setting baud rate?
--------------------------------------------------
STATUS: success
TIME: 2.05s

ANSWER:
### 🧠 Hardware Insight: UART DLAB Behavior
**DLAB** (Divisor Latch Access Bit) is **Bit 7** of the **UxLCR** register.

**Reasoning Chain:**
- **DLAB = 1:** The CPU can access the Baud Rate Divisors (**UxDLL** and **UxDLM**).
- **DLAB = 0:** The CPU can access the **UxTHR** (Transmit) and **UxRBR** (Receive) registers.

**Hardware Result:**
- **Critical Error:** If DLAB is not cleared (set to 0) after initialization, any attempt to send data will instead overwrite the baud rate divisor.
- **Result:** Communication will fail completely.

QUESTION 16: Explain ADC noise behavior in LPC2148
--------------------------------------------------
STATUS: success
TIME: 2.08s

ANSWER:
### 🧠 Hardware Insight: High Impedance ADC Input
The LPC2148 ADC uses a switched-capacitor successive approximation architecture.

**Reasoning Chain:**
- **Charging:** The internal sample-and-hold capacitor must charge to the input voltage during the sampling window.
- **Impedance:** If the source impedance is high (>10kΩ), the capacitor cannot charge fully in time.

**Hardware Result:**
- **Result:** Inaccurate, noisy, or fluctuating readings that do not match the actual DC input level.
- **Fix:** Use a low-impedance source or an op-amp buffer before the ADC pin.
